# 飞书博客文档格式规范

## 概述

本博客系统从飞书知识库读取博客文章。只需在飞书文档中按照规范格式编写，网站会自动同步更新。

## 文档结构

### 方式一：YAML Frontmatter（推荐）

在文档开头添加 YAML 元数据：

```markdown
---
title: 文章标题
category: 技术
tags: [React, TypeScript, 前端]
date: 2024-12-15
cover: https://images.unsplash.com/photo-xxx?w=800&q=80
readTime: 8
excerpt: 文章摘要，不超过150字
---

## 正文标题

这里是文章正文...

### 子标题

更多内容...
```

### 方式二：自动提取（简单）

如果不添加 frontmatter，系统会自动从文档中提取：

- **标题**：第一个一级标题（# 标题）
- **摘要**：第一段文本的前100字
- **日期**：文档创建日期
- **阅读时间**：根据字数自动计算

## 必需字段

| 字段 | 说明 | 示例 |
|------|------|------|
| title | 文章标题 | React 18 新特性详解 |
| category | 文章分类 | 技术 / 生活 / 旅行 |
| date | 发布日期 | 2024-12-15 |

## 可选字段

| 字段 | 说明 | 默认值 |
|------|------|--------|
| tags | 标签数组 | [] |
| cover | 封面图片URL | 根据分类使用默认封面 |
| readTime | 阅读时长（分钟） | 根据字数计算 |
| excerpt | 文章摘要 | 从正文提取 |

## 分类与封面图

系统为不同分类提供了默认封面图：

- **技术**：代码相关图片
- **生活**：生活场景图片
- **旅行**：风景图片
- **其他**：通用封面图片

你也可以通过 `cover` 字段指定自定义封面图。

## 内容格式支持

飞书文档支持以下 Markdown 元素：

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 文本样式

```markdown
**粗体文本**
*斜体文本*
`行内代码`
```

### 列表

```markdown
- 无序列表项
- 另一项

1. 有序列表项
2. 第二项
```

### 代码块

````markdown
```javascript
function hello() {
  console.log('Hello, World!');
}
```
````

### 图片

```markdown
![图片描述](图片URL)
```

### 链接

```markdown
[链接文本](https://example.com)
```

## 飞书知识库设置

### 1. 创建知识库

1. 打开飞书，进入「知识库」
2. 点击「创建知识库」
3. 命名为「个人博客」或你喜欢的名称

### 2. 创建博客文件夹

1. 在知识库根目录创建「博客」文件夹
2. 记录文件夹的 Token（从 URL 获取）

### 3. 配置环境变量

将知识库和文件夹的 Token 填入项目 `.env` 文件：

```env
VITE_FEISHU_WIKI_TOKEN=你的知识库Token
VITE_FEISHU_BLOG_FOLDER_TOKEN=博客文件夹Token
```

### 4. 编写博客

在「博客」文件夹中创建新文档，按照上述格式编写即可。

## 最佳实践

1. **封面图片**
   - 使用高清晰度图片（推荐 1920x1080）
   - 使用 Unsplash、Pexels 等免费图库
   - 保持图片风格统一

2. **摘要**
   - 控制在 100-150 字
   - 概括文章核心内容
   - 吸引读者点击

3. **标签**
   - 每篇文章 3-5 个标签
   - 使用常见技术术语
   - 保持标签一致性

4. **分类**
   - 使用预定义分类
   - 一篇文章一个分类
   - 可在代码中扩展分类列表

5. **SEO 优化**
   - 标题包含关键词
   - 摘要准确描述内容
   - 使用语义化的标题层级

## 示例文档

```markdown
---
title: React Hooks 最佳实践
category: 技术
tags: [React, Hooks, 前端]
date: 2024-12-20
cover: https://images.unsplash.com/photo-1633356122544-f134324a6cee?w=800&q=80
readTime: 10
excerpt: 深入理解 React Hooks 的使用场景和最佳实践，避免常见陷阱。
---

## React Hooks 最佳实践

React Hooks 彻底改变了我们编写 React 组件的方式...

### 为什么使用 Hooks

Hooks 让我们能够在函数组件中使用状态和其他 React 特性...

### 常用 Hooks

1. **useState** - 状态管理
2. **useEffect** - 副作用处理
3. **useContext** - 上下文访问

### 最佳实践

- 保持 Hooks 在顶层调用
- 只在 React 函数中使用
- 使用 ESLint 插件检查规则

### 总结

遵循这些最佳实践，你的代码将更加清晰和可维护。
```

## 技术细节

### 缓存机制

系统默认缓存飞书 API 响应 10 分钟，减少 API 调用次数。你可以：

1. 修改 `.env` 中的 `VITE_CACHE_DURATION` 调整缓存时长
2. 调用 `clearCache()` 清除缓存
3. 重启代理服务器清除所有缓存

### 降级策略

当飞书 API 不可用时，系统会自动降级到静态数据：

1. 优先使用缓存数据
2. 缓存过期则使用静态数据
3. 保证网站始终可用

### API 限流

飞书 API 有调用频率限制：

- 租户访问令牌：100次/秒
- 文档读取：根据具体 API 文档

使用缓存可以有效避免限流问题。

## 常见问题

### Q: 文章没有出现在网站上？

A: 检查以下几点：
1. 文档是否在正确的文件夹内
2. frontmatter 格式是否正确
3. 查看浏览器控制台是否有错误

### Q: 如何更新已发布的文章？

A: 直接在飞书文档中编辑，等待缓存过期（默认10分钟）或手动清除缓存即可。

### Q: 支持草稿功能吗？

A: 可以在飞书中创建「草稿」文件夹，将未完成的文章移入其中，只将「博客」文件夹连接到网站。

### Q: 如何修改分类？

A: 在 `blogs.ts` 中修改 `categories` 数组即可添加新分类。
